二、用户使用与模型响应流程图

1. 用户提问

用户在 Web、管理端或业务系统中输入问题，并选择目标知识库。

→

2. FastAPI 接收请求

RAG Gateway 校验请求参数，读取配置，进入知识库服务编排流程。

→

3. 定位知识库

KnowledgeBaseService 查询 JSON 元数据，拿到知识库对应的 vector_store_id 和文档映射。

→

4. OpenAI 托管检索

OpenAI Backend 调用 File Search / Vector Store 检索，获得相关文档片段、分数和引用信息。

5. 组装上下文

系统将命中的 chunks、citations 和用户问题组合为模型输入上下文。

→

6. 模型生成答案

OpenAI 模型基于检索上下文生成回答，并保留引用来源。

→

7. 返回结果

FastAPI 将 answer、citations、命中文档信息返回给用户或业务系统。

图中需要表达的重点

OpenAI Vector Stores 是主向量库，OpenAI File Search / RAG 检索是主检索路径。

FastAPI 主要承担网关、业务编排、元数据维护和后端适配；本地 RAG 管线是 POC/开发模式能力，可作为后续 hybrid 后端的一部分。
