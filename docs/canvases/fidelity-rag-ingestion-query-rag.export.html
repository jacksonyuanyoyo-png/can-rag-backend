本地 RagPipeline
工程内已落地（入库）

RagPipeline：chunk → embed → JsonVectorStore；由 KnowledgeBaseService.index_document 读取文本后触发。

  • 源文件 → LOCAL_UPLOAD_ROOT
  • read_text → RagPipeline.index_document
  • TextChunker.split（RAG_CHUNK_SIZE / OVERLAP）
  • HashEmbeddingService.embed（每段）
  • JsonVectorStore.upsert_document
对话检索
  • 用户 Query
  • HashEmbeddingService.embed(query)
  • JsonVectorStore.search + SimpleReranker
  • SearchHit → 拼装 Prompt → LLM（业务层对接）

图中需要表达的重点

OpenAI 路径中步骤 5 的解析、分段、向量化不在本仓库显式实现，由 Vector Store 托管流水线完成；本仓库负责 REST 编排与 KnowledgeBaseRepository 元数据。

视觉规范与 fidelity-rag-layered-and-flow 一致：浅灰背景、白底容器、虚线分区、浅蓝序号徽章、白卡片步骤块与灰色箭头。
